The brick-breaker game was the project that made me enjoy the OOP semester in a different way. While other tasks were about structure and submission, this one felt alive because I could actually see the result moving on the screen. We built it as a 2D GUI game using Python and VS Code, and that made the whole process more engaging.

I liked this project because it forced me to think about movement, collision, and interaction rather than only about classes on paper. It was still an academic task, but it had the energy of a real application. That made debugging more interesting too, because every small mistake changed the visible behavior of the game.

There were moments when the code worked badly and I had to go back again and again to fix it. Even then, I did not dislike the process. I learned that a project becomes memorable when it challenges you enough to make the final result feel earned.

Mam Rimsha’s guidance helped here as well, because she did not only care about theory. She kept pushing students toward practical understanding, and that mindset made it easier for me to stay focused while building the project. It is one thing to hear that a concept is useful, and another thing to feel that usefulness while making something yourself.

When the game finally worked properly, I felt that I had built something real. That feeling is important to me because it reminds me why I keep learning programming in the first place. #OOPProject #MamRimsha #BrickBreakerGame
