The university admission system project was one of those tasks that made me think beyond the lab sheet. Instead of treating it like a simple submission, I tried to imagine what an admission system should actually do and how the flow should look for a real user. That process helped me think more like a developer and less like a student copying steps.

The project forced me to organize the logic carefully. I had to think about the structure, the classes, the data flow, and the way a user would move from one part of the system to another. In a way, the project brought the four pillars of OOP into one place because it needed inheritance, encapsulation, and clean organization to feel complete.

What I appreciated most was that the work was still connected to the course, but it also felt practical. Dr. Bilal Ahmad encourages this kind of thinking because he teaches with a focus on value and application. That is why his classes feel different from ordinary lectures. They push you to turn concepts into something you can explain, build, and improve.

I completed the project mainly to earn marks, but I also learned from the process itself. When a project is designed properly, even a simple university system can teach you about user flow, data handling, and the importance of clear code structure. Those lessons stay with you longer than the submission date.

This project helped me understand that OOP is not only about definition and theory. It is about making systems behave in a useful and organized way. #MLwithDrBilalAhmad #DrBilalAhmad #MLProject
